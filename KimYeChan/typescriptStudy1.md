# Typescript를 쓰는 이유

## 타입 안정성

- 코드에 버그 줄어들고
- 런타임 에러(실행할 때 발생하는 에러) 줄어들고
- 생산성 늘어남

타입스크립트의 보호장치는 타입스크립트가 자바스크립트로 변환되기 전에 실행된다.

타입스크립트 코드에 오류가 있다면 자바스크립트 코드로 컴파일 되지 않는다.

타입스크립트의 보호장치는 타입 추론 때문에 생긴다.(타입을 지정해주지 않아도 타입을 추론한다)

## 타입을 쓰는 두가지 방법

1. 명시적 타입
    
    let 변수명 : 타입 = ~
    
    ex ) let a : string = ‘hi’
    
2. 암시적 타입
    
    let a = ‘hi’ → 자바스크립트 쓰듯이(타입스크립트가 알아서 추론함)
